# Code Of Conduct

Dogtap aims to be a practical, respectful engineering project.

## Expected Behavior

- Be direct, factual, and constructive.
- Keep discussion focused on the technical problem and user impact.
- Assume good intent, but make risks and tradeoffs explicit.
- Respect privacy: do not post secrets, customer data, raw production telemetry,
  or private adoption details.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing private information without consent.
- Repeatedly derailing technical discussion.
- Encouraging unsafe use of Dogtap in production telemetry or
  security-sensitive paths beyond documented boundaries.

## Enforcement

Maintainers may edit or remove comments, close issues, restrict participation,
or block users when behavior harms the project or its users.

If a report involves a security issue or private data, follow `SECURITY.md`
instead of opening a public issue.
