# G1 Fixture Evidence Harness

## Status

Current status: G1 passed for the first product slice on 2026-05-08.

The original bundled payloads under `fixtures/` remain smoke fixtures, but
reviewed and sanitized real-evidence fixtures now exist for RUM, APM, and OTLP.
Logs evidence is generated by the local logs capture harness. Keep run-local
raw evidence under `testdata/g1-evidence/latest/` and promote only reviewed
sanitized artifacts.

## Artifact Layout

Capture scripts write generated evidence under:

```text
testdata/g1-evidence/latest/
|-- logs/
|   |-- response-json.json
|   |-- response-text.json
|   |-- response-gzip.json
|   |-- log-gzip.json.gz
|   `-- events.json
|-- rum/
|   |-- events.json
|   `-- README-not-run.txt
|-- apm/
|   |-- events.json
|   `-- README-not-run.txt
`-- otlp/
    |-- events.json
    `-- README-not-run.txt
```

The `latest/` directory is intentionally ignored because it is run-local
evidence. Promote reviewed artifacts into `fixtures/` only after confirming
they came from the declared capture command.

## Commands

Run the logs fixture capture, which uses only local `curl`, `gzip`, and Dogtap:

```bash
scripts/fixtures/capture-logs.sh
```

Run all harness scripts:

```bash
scripts/fixtures/capture-all.sh
```

The all-script does not install dependencies. Scripts that need SDK packages
write `README-not-run.txt` with exact setup commands when dependencies are
missing.

## RUM Browser SDK Evidence

Goal: capture a payload emitted by `@datadog/browser-rum` from a minimal browser
app, routed through a same-origin local proxy to Dogtap.

Expected commands:

```bash
npm --prefix testdata/rum-browser install
scripts/fixtures/capture-rum-browser.sh
```

Expected artifact:

```text
testdata/g1-evidence/latest/rum/events.json
```

Passed evidence shows at least one Dogtap event with:

- `source=rum`
- `service=web-frontend`
- `env=local`
- `version=g1-fixture`
- session or view identity
- user/account/workspace/case context for the positive flow
- redaction expectations for authorization, tokens, and email-bearing payloads

## Logs Evidence

Goal: cover Datadog logs HTTP-style JSON, text, and gzip request bodies through
Dogtap.

Expected command:

```bash
scripts/fixtures/capture-logs.sh
```

Expected artifacts:

```text
testdata/g1-evidence/latest/logs/events.json
testdata/g1-evidence/latest/logs/log-gzip.json.gz
```

Passed evidence shows:

- JSON log normalized service/env/version/trace/span/route fields
- text log accepted as message payload
- gzip payload decoded by Dogtap
- sensitive headers and query tokens masked in stored request metadata

## Promotion Checklist

Before changing any fixture metadata from `smoke` to `g1-real-evidence`:

- Record the exact command, date, local OS/runtime versions, and script commit.
- Keep captured telemetry local; do not call Datadog intake APIs.
- Confirm no real user, customer, account, or production payload data exists.
- Store expected normalized fields and redaction expectations next to the
  promoted fixture.
- Update `specs/000-product/tasks.md` and `specs/000-product/research.md` with
  evidence notes.
